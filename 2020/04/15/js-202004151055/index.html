js事件详解+正则

Sakuracatta · 2020-4-15 · 次阅读

事件详解

1鼠标事件

1.1 基础

事件源：事件发生在谁身上

事件类型：用户的行为

事件处理程序：事件发生后的响应

1.2 事件的添加与移除
  1. 添加单个事件的方法：
div.onclick=function(){
  alert(1)//不会弹出，因为下面的把它覆盖了，为了避免这种情况，则可以用添加多个事件时用
}
div.onclick=function(){
  alert(2)//会弹出，
}
  1. （现代浏览器）添加多个事件的方法：

对象.addEventListener(事件类型，事件处理程序/函数名，布尔值)

div.addEventListener('click', function(){
  alert(1)
}, false)
//或者：
div.addEventListener('click', move, false)
function move(){
  alert(1)
}

​ 2.1 移除事件

对象.removeEventListener(事件类型，事件处理程序/函数名，布尔值)

div.removeEventListener('click', move, false)
  1. IE低版本中的事件添加与移除

添加：对象.attachEvent ( on事件类型，事件处理程序/函数名)

移除：对象.detachEvent ( on事件类型，事件处理程序/函数名)

注意：只有这两个参数，添加和移除的次序都是倒序执行的

现代浏览器与IE添加事件的区别 现代浏览器 IE低版本
参数个数不同 3 2
事件类型不同 不加on 加on
执行顺序不同 顺序执行 倒序执行
this所指对象不同

3 鼠标滚动事件

比较 事件 获取滚动的方向 值
谷歌/IE浏览器 onmousewheel e.wheelDelta 上120，下-120
火狐浏览器 DOMMouseScroll e.detail 上-3， 下3

兼容函数

function mousewheel(obj,upfun,downfun) {
        //判断浏览器是否支持attachEvent，支持为IE，不支持为谷歌，火狐
        if (obj.attachEvent){
            obj.attachEvent(`onmousewheel`,wheel);
        }else {
            obj.addEventListener(`mousewheel`,wheel);
            obj.addEventListener(`DOMMouseScroll`,wheel);
        }
        function wheel(e){
            //将事件对象做兼容，谷歌火狐为e，IE为window.event
            var ev=e||window.event;
            flag = true;
            //阻止浏览器默认的动作，例如在盒子上滚轮向下，整个页面也会滚动，因为浏览器默认的动作，所以需要阻止
            if(ev.preventDefault){
                ev.preventDefault();  //谷歌，火狐，ie高版本
            }else{
                ev.returnValue=false;  //ie低版本
            }
            //判断鼠标滚轮的方向
            if (ev.wheelDelta==-120||ev.detail==3){      //滚轮向下
                if (flag){
                    flag=false;
                    downfun();
                }
            }else if(e.wheelDelta==120||e.detail==-3) {  //滚轮向上
                if (flag){
                    flag=false;
                    upfun();
                }
            }else {
                return;
            }
        }
    }

4 事件流

当一个事件触发的时候，当前元素的父元素，乃至整个页面都会以一定的顺序来响应这个事件，响应的顺序叫做是事件流

4.1 冒泡型事件流

从当前元素一级一级向上触发，触发到html

<div class="par">
    <div class="son"></div>
</div>
<script>
  son.onclick = function() {
      console.log('son冒泡')
  }
  par.onclick = function() {
      console.log('par冒泡')
  }
  body.onclick = function() {
      console.log('body冒泡')
  }
  html.onclick = function() {
      console.log('html冒泡')
  }
</script>    
<--例如：只点击son时，会从son到html根元素一直触发事件，console的结果是：son冒泡、par冒泡、body冒泡、html冒泡-->
4.2 捕获型事件流

从根元素一级一级向下触发，到当前元素

如何添加捕获型事件流？在addEventListener里，把 默认的false改成true

兼容性：ie只有冒泡型事件流 ，没有捕获型事件流

son.addEventListener('click', function(){
    console.log('son捕获')
}, true)
par.addEventListener('click', function(){
    console.log('par捕获')
}, true)
body.addEventListener('click', function(){
    console.log('body捕获')
}, true)
html.addEventListener('click', function(){
    console.log('html捕获')
}, true)
//例如：只点击par时，事件流会从html一路向下触发到当前目标事件源，console的结果是：html冒泡、body冒泡、par冒泡
4.3 阻止冒泡型事件流的传递
son.onclick = function(e) {
    // 阻止事件流的传递，不是阻止事件流
    if(e.stopPropagation){
        e.stopPropagation()//高级浏览器，IE不兼容
    }else{
        e.cancelBubble = true//IE，IE里面都是属性，现代浏览器都是方法
    }    
    // 注意：这是从son开始阻止传递，如果给par阻止，son不阻止，则单击son时会触发son和par
    console.log('son冒泡')
}

在运用中，父元素与子元素来回切换时仍然会有事件流的传递

4.4 事件委派

what?把子元素要添加的事件委派给父元素，利用的是冒泡型事件流的特性，点击obj目标事件源会向上触发事件

when?1.给新添加的元素加事件 2.优化程序，用于大量事件的添加 3. ajax时需要用

document.body.onclick=function(e){
    var obj = e.target
    if(obj.nodeName == 'DIV'){
        obj.style.background='blue'
    }
}
//例如：把给div盒子的点击事件委派给document.body，利用冒泡事件流传递给div

注意：onmouseenter和onmouseleave没有事件流，所以也没有事件委派

5 cookie

1. h5之前是cookie，h5之后是本地存储
2. cookie是键值对的结构
3. 电脑只允许写入4k的cookie，超过4k时有的浏览器随机删除，有的浏览器删除以前的
4. cookie不能跨浏览器访问,例如，在谷歌浏览器里设置的cookie，在火狐等其他浏览器里是没有的
5.1 设置cookie
document.cookie = 'name=z3'
document.cookie = 'age=18'
// 访问所有的cookie：console.log(document.cookie)，获取回的是字符串
//以上是临时性的，生命周期是浏览器的打开到关闭的时间
//以下设置一定时间的cookie
document.cookie = 'name=zhangsan; expires='+now.toGMTString();//now.toGMTString()表示需要设置的到期的时间

注意：多个cookie之间都是由‘; ’分号空格隔开，单个cookie的键和值是由‘=’分隔开，

这样获取cookie时可以先用字符串的split()方法进行分割成数组var arr = str.split(‘; ’)；var brr = str.split(‘=’)

key不能重复，value可以重复

正则表达式

1 概念

1.1 正则表达式是普通字符与特殊字符组成的模式字符串

1.2 正则在php中经常用，要好好学

2 正则的方法

2.1 test()

正则是否匹配到，返回一个布尔值

2.2 exec()

exec() 方法用于检索字符串中的正则表达式的匹配。
返回值是一个数组，但是此数组的内容和正则对象是否是全局匹配有着很大关系

//非全局匹配
var str = 'hehuan'
var reg = /h/
console.log(reg.exec(str))//["h", index: 0, input: "hehuan"]
//全局匹配见下面精讲

3 精讲

3.1 全局匹配

模式修正符g是全局匹配，自动刷新，匹配过的的不匹配

var str = ‘asdfasdf’
var reg = /a/g
console.log(reg.test(str),reg.lastIndex) //true 1 从0开始匹配，匹配到了第一个
console.log(reg.test(str),reg.lastIndex) //true 5 从上次匹配到的位置继续往后，找到第5个
console.log(reg.test(str),reg.lastIndex) //false 0 从上次匹配到的位置继续往后，第5个后面没有a，则没找到
console.log(reg.test(str),reg.lastIndex) //true 1 没找到又会从0位置开始找

原理：加模式修正符g会改变正则对象的lastIndex属性的值，即找到的位置

3.2 全局匹配与非全局匹配的区别

非全局匹配：

作用和match()函数是一样的，只能够在字符串中匹配一次，如果没有找到匹配的字符串，那么返回null，否则将返回一个数组

全局匹配

此函数返回值同样是一个数组，并且也只能够在字符串中匹配一次。不过此时，此函数一般会和lastIndex属性匹配使用，此函数会在lastIndex属性指定的字符处开始检索字符串，当exec()找到与表达式相匹配的字符串时，在匹配后，它将lastIndex 属性设置为匹配字符串的最后一个字符的下一个位置。可以通过反复调用exec()函数遍历字符串中的所有匹配，当exec()函数再也找不到匹配的文本时，它将返回null，并把lastIndex 属性重置为0。

3.3 原子

原子 描述（一个原子只能匹配一个字符） 原子 描述
\d 用来匹配0-9的数字 \D 用来匹配除了0-9数字之外的所有字符
\w 用来匹配数字字母下划线：0-9 a-z A-Z _ \W 用来匹配除了数字字母下划线的所有字符
\s 用来匹配空白字符：空格、\n、\t、\f、\r、\v \S 用来匹配除了空白字符的所有字符
. 除了换行之外，所有的都能匹配
^ 匹配边界，一行的开始位置 $ 匹配边界，一行的结束位置

3.4 原子表

3.4.1 原子表的引源

1. 或'|'的含义
var reg = /ab|cb/     //表示匹配ab或cb,不是a，b或c，b
2. 如果要表示a，b或c，b，则加括号，如下：
var reg = /a(b|c)b/
3. 当有扩号的时候会生成原子分组,在匹配的时候，会将扩号里面匹配到的内容会压入内存，等待使用
4. 使用的过程就叫反向引用，例如：
var reg = /a(b|c)b(\d)\1\2/     //有俩个括号则有俩个原子分组
var str = 'acb9c9'    //匹配到了，因为acb扩号，第一个扩号保存下了c，第二个扩号保存下了9，所以后面\1引用到了c，\2引用到了9,正好匹配到
5. 取消反向引用，如下： 
var reg = /a(?:b|c)b(\d)\1///会把第一个扩号的引用取消掉，\1匹配到的是第二个扩号，因为第一个已经取消反向引用

3.4.2 自定义的原子

/[1-5]/匹配1-5的数字  /[a-c]/匹配a-c之内的数字
/[1-59]/匹配1-5和9这几个数组：1/2/3/4/5/9
取反：/[^1-5]/除了1-5外的所有字符
var reg = /\d{3,5}/ //匹配3-5次\d
var reg = /\d{3,}/ //匹配最少3次\d
var reg = /\d{0,5}/ //匹配0-5次\d
原子 描述 原子 描述
+ 匹配1次到多次 ？ 匹配0次或1次
* 匹配0次或多次

3.4.3 吝啬与贪婪

例如 str='123456b'
// 贪婪匹配，尽可能多的匹配 默认 var reg = /123\d{2,4}/; reg.exec(str)结果是123456
// 吝啬匹配，尽可能少的匹配 ？    var reg = /123\d{2,4}?/; reg.exec(str)结果是12345
// 在匹配成功的前提下再吝啬匹配
